# Travel AI Agent — 项目速读笔记

## 1. 项目整体介绍

**定位**: 基于 DeepSeek 大模型的智能旅行规划 Agent，提供从目的地介绍 → 行程生成 → 美食住宿推荐 → 交通规划 → 动态调整的全流程服务。

**核心职责**:
- 接收用户旅行需求（目的地、日期、人数、预算、老人/小孩数量等）
- 自动查询天气、计算城市间距离
- 通过 LLM 推理引擎生成个性化旅行规划
- 结果保存为 Markdown 文件，接口返回文件路径

**技术栈**:

| 层 | 技术 |
|---|------|
| Web 框架 | FastAPI + Uvicorn |
| LLM | DeepSeek Chat API (openai 兼容接口) |
| 数据库 | SQLite (aiosqlite 异步) |
| 天气数据 | Open-Meteo 免费 API (MCP 协议封装) |
| 异步 HTTP | httpx |
| 数据校验 | Pydantic v2 |

**端口**: 8000 (可配置 `SERVER_PORT`)

## 2. 项目架构

```
main.py (FastAPI 入口)
  │
  ├── GET  /           HTML 前端表单
  ├── POST /api/plan   旅行规划（核心）
  ├── POST /api/chat   通用对话
  ├── POST /api/chat/stream  SSE 流式
  ├── GET  /api/health /tools /sessions /preferences
  │
  └── agent/core.py: TravelAgent (总调度)
        │
        ├── StrategySelector     自动选择推理策略
        ├── agent/llm.py         DeepSeek 客户端 (带重试)
        ├── agent/memory/         SQLite 记忆 (会话+偏好)
        ├── agent/mcp/            MCP 工具协议
        │     ├── provider.py    工具注册/调用
        │     ├── weather.py     天气查询 (Open-Meteo)
        │     ├── distance.py    城市距离计算
        │     └── city_data.py   50个城市坐标 (共享)
        ├── agent/reasoning/     6种推理引擎
        │     ├── react.py       ReAct (思考→行动→观察)
        │     ├── cot.py         Chain of Thought
        │     ├── tot.py         Tree of Thoughts
        │     ├── mcts.py        Monte Carlo Tree Search
        │     ├── reflexion.py   Reflexion (自改进)
        │     └── decompose.py   任务分解 (DAG依赖排序)
        └── agent/planner/       旅行规划模块
              ├── destination.py 目的地介绍
              ├── itinerary.py   行程生成 + 约束构建
              ├── food_stay.py   美食 + 住宿推荐
              ├── transport.py   交通规划
              └── adjuster.py    动态调整引擎
```

### 核心机制: 策略自动选择

```
用户输入 → StrategySelector.select()
  ├── 含 "规划/行程/攻略"     → decompose (任务分解)
  ├── 含 "最优/路线/方案对比"  → tot (思维树)
  ├── 含 "改进/调整/重新规划"  → reflexion (反思)
  ├── 含 "分析/比较/推荐理由"  → cot (思维链)
  └── 默认                    → react (推理+工具调用)
```

用户可通过 `strategy` 参数强制覆盖。

### 核心机制: /api/plan 执行流程

```
请求 → TravelAgent.plan_travel()
  │
  ├── Step 1: 天气查询 (MCP weather tool, 非关键, 降级)
  ├── Step 2: 目的地介绍 (LLM 单次调用, ~15s)
  ├── Step 3: 行程生成 (LLM 单次调用, ~25s, 最耗时)
  ├── Step 4-6: 美食 + 住宿 + 交通 (asyncio.gather 并行, ~20s)
  ├── Step 7: 动态调整 (LLM, ~7s)
  ├── Step 8: 生成 Tips
  │
  └── 保存 outputs/{目的地}_{日期}_{时间戳}.md → 返回文件路径
```

总耗时约 60-90 秒，瓶颈在 LLM 调用延迟。

### 核心机制: 动态约束系统

`build_constraints()` 根据实际参数生成约束条件注入 LLM prompt:

| 参数 | 触发条件 | 约束示例 |
|------|---------|---------|
| `num_elderly > 0` | 有老人 | 节奏放缓、每2h休息、电梯优先 |
| `num_children > 0` | 有小孩 | 亲子景点、午休、避免危险活动 |
| `num_travelers >= 5` | 多人 | 预订餐厅/门票、包车 |
| `travel_mode` | 出行方式 | 自驾: 停车+驾驶≤3h; 高铁: 接驳 |
| `weather 降水 > 60%` | 雨天 | 户外改室内备选 |
| `weather 温度 > 35` | 高温 | 避免中午户外 |

### 关键配置

```python
# config.py
DEEPSEEK_API_KEY    # 必须
LLM_MAX_RETRIES = 3      # 指数退避重试
LLM_TIMEOUT = 120        # LLM 请求超时
WEATHER_CACHE_TTL = 1800 # 天气缓存 30 分钟
MAX_REACT_STEPS = 10     # ReAct 最大步数
MCTS_ITERATIONS = 50     # MCTS 搜索次数
```

## 4. 核心业务流程

### 主流程: 生成旅行规划

```
用户提交表单
  │
  ▼
FastAPI POST /api/plan (main.py:160)
  │ 校验 TravelPlanRequest (Pydantic)
  ▼
TravelAgent.plan_travel() (core.py:163)
  │
  ├─► mcp_provider.call_tool("weather")  ──► Open-Meteo API
  │     └─ 失败 → 降级, weather_data=None
  │
  ├─► get_destination_intro() ──► DeepSeek LLM
  │
  ├─► generate_itinerary()
  │     ├─ build_constraints()   根据参数生成约束文本
  │     ├─ build_weather_section() 格式化天气
  │     └─ DeepSeek LLM (含约束 + 天气)
  │
  ├─► asyncio.gather(               ← 并行
  │     recommend_food(),           DeepSeek
  │     recommend_accommodation(),  DeepSeek
  │     plan_transport(),           DeepSeek
  │   )
  │
  ├─► dynamic_adjuster.adjust()   ──► DeepSeek
  │
  ├─► _generate_tips()
  │
  └─► 写入 outputs/*.md → 返回 {status, file, session_id}
```

### 对话流程: /api/chat

```
用户消息 → StrategySelector.select()
  │
  ├── "react":    ReActEngine.reason()
  │     └─ Loop: LLM→解析 Thought/Action→调用MCP工具→Observation→循环
  │
  ├── "cot":      CoTEngine.reason()
  │     └─ LLM 逐步推理, 无工具调用
  │
  ├── "tot":      ToTEngine.reason()
  │     └─ 生成 breadth×depth 候选分支 → 评估 → 选最优 → 润色输出
  │
  ├── "mcts":     MCTSEngine.reason()
  │     └─ Selection→Expansion→Simulation→Backpropagation ×50 轮
  │
  ├── "reflexion":ReflexionEngine.reason()
  │     └─ 生成→评估→反思→修正, 最多 3 轮, 评分≥8.5 提前退出
  │
  └── "decompose":TaskDecomposer.execute()
        └─ LLM 拆解为子任务 DAG → 拓扑排序 → 逐个执行 → 综合输出
```

### MCP 工具调用链

```
agent/mcp/provider.py
  ├─ list_tools() → 返回工具描述 (注入 LLM prompt)
  └─ call_tool(name, args)
       ├─ "weather"  → WeatherTool.execute(city, date)
       │     ├─ city_data.lookup_coords() 查找经纬度
       │     ├─ httpx → Open-Meteo API
       │     └─ json.dumps() 返回结果
       └─ "distance" → DistanceTool.execute(from_city, to_city)
             ├─ city_data.lookup_coords() ×2
             └─ city_data.haversine_distance() 计算
```

## 5. 快速上手

### 启动命令

```bash
# 安装依赖
pip install -r requirements.txt --break-system-packages

# 配置 API Key
export DEEPSEEK_API_KEY="sk-xxx"

# 启动 (开发模式)
uvicorn main:app --reload --port 8000

# 或直接运行
python main.py
```

### 建议代码阅读顺序

1. **`config.py`** (36 行) — 了解所有配置项和默认值
2. **`models/schemas.py`** — 理解 `TravelPlanRequest` 字段含义
3. **`main.py`** — 看 API 端点定义, 关注 `/api/plan` (行 160-187)
4. **`agent/core.py`** — `StrategySelector` (行 41-70) 和 `TravelAgent.plan_travel()` (行 163) 是核心
5. **`agent/mcp/city_data.py`** — 城市坐标和距离计算公用函数
6. **`agent/mcp/weather.py`** — `WeatherTool.execute()` 天气查询实现
7. **`agent/planner/itinerary.py`** — `build_constraints()` 约束构建逻辑
8. **`agent/reasoning/react.py`** — 最常用的推理引擎 (思考→行动→观察循环)
9. **`agent/memory/__init__.py`** — SQLite 表结构和 `build_context()` 上下文构建

### 如何添加新功能

**添加新 MCP 工具** (如: 酒店搜索):
1. 在 `agent/mcp/` 下新建 `hotel.py`, 实现 `class HotelTool` (含 `name`, `description`, `parameters`, `execute()`)
2. 在 `agent/mcp/provider.py` 的 `_register_builtin_tools()` 中注册
3. 工具自动出现在 `/api/tools` 和 LLM 提示词中

**添加新推理策略**:
1. 在 `agent/reasoning/` 下新建文件, 实现 `async reason(task, ...) -> dict`
2. 在 `agent/reasoning/__init__.py` 导出
3. 在 `agent/core.py` 的 `StrategySelector.select()` 添加匹配规则
4. 在 `TravelAgent.chat()` 的 if/elif 链添加分支

**修改前端表单**:
- 编辑 `main.py` 中 `@app.get("/")` 返回的 HTML 字符串
- 如需新字段, 同步修改 `models/schemas.py` 的 `TravelPlanRequest`
- 同步修改 `agent/core.py` 的 `plan_travel()` 和各 planner 函数签名
